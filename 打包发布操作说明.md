# 打包发布操作说明

## 1. 推荐构建方式

当前项目优先使用 `onedir` 调试与发布验证。

原因：

- 当前已知问题与 `_internal` 目录中第三方包收集不完整直接相关。
- `onedir` 最容易核对 `imblearn/sklearn/scipy/PySide6/torch` 的目录、`.pyd`、`.dll` 和数据文件。
- 只有 `onedir` 自检和安装自检都通过后，再考虑切换 `onefile`。

## 2. 构建前准备

建议在干净环境中执行：

```bat
python -m pip install -r requirements.txt
python -m pip install -r requirements-build.txt
```

建议先删除旧构建产物，再重新生成：

```bat
rmdir /s /q build
rmdir /s /q dist
rmdir /s /q dist_installer
```

如果不想手工删除，也至少要确保旧的 `dist/FraudShield/_internal` 没有残留过期文件。

## 3. PyInstaller 打包

已更新 `FraudShield.spec`，关键修复如下：

- 项目根目录改为 `Path(__file__).resolve().parent`
- 不再依赖当前工作目录
- 显式 `collect_all()`：
  - `imblearn`
  - `sklearn`
  - `joblib`
  - `numpy`
  - `scipy`
  - `pandas`
  - `matplotlib`
  - `PySide6`
  - `cryptography`
  - `torch`
- 额外保留 `collect_dynamic_libs("torch")`

执行方式：

```bat
build_release.bat
```

或者直接执行：

```bat
python -m PyInstaller --noconfirm --clean FraudShield.spec
```

## 4. 打包后必须先做 onedir 自检

先不要急着做 Inno Setup。

先在 `dist/FraudShield` 下检查：

1. `FraudShield.exe` 是否存在
2. `_internal` 是否存在
3. `_internal/imblearn` 是否存在
4. `_internal/sklearn` 是否存在
5. `_internal/scipy` 是否存在
6. `_internal/PySide6` 是否存在
7. `assets/styles/dark_finance.qss` 是否存在
8. `data/demo/sample_demo.csv` 是否存在
9. `examples/sample_inference.csv` 是否存在

然后直接运行：

```bat
dist\FraudShield\FraudShield.exe
```

验证要求：

- 主界面能先打开
- 打开“模型对比”页不应导致程序退出
- 点击“开始对比”时，如果依赖仍有问题，应该只报功能错误，不应把主程序一起带崩
- 启动后能在用户数据目录看到：
  - `logs/startup.log`
  - `logs/error.log`

## 5. Inno Setup 打包

确认 onedir 自检通过后，再执行：

```bat
build_installer.bat
```

当前安装脚本 `installer/FraudShield.iss` 采用：

- 源目录：`..\dist\FraudShield`
- 递归拷贝整个 `onedir` 目录

这符合当前项目结构，不建议在 Inno Setup 阶段手工裁剪 `_internal`。

## 6. 安装后验证顺序

安装完成后，按下面顺序验证：

1. 在安装目录直接双击 `FraudShield.exe`
2. 检查主窗口是否显示
3. 检查日志目录是否生成 `startup.log` 和 `error.log`
4. 打开以下页面确认不会在进入时崩溃：
   - 数据导入
   - 数据分析
   - 模型对比
   - 模型训练
   - 模型评估
   - 风险推理
5. 在“模型对比”页点击“开始对比”
6. 在“模型训练”页点击“开始训练”
7. 在“模型评估/风险推理”页验证模型文件可加载

## 7. 失败时优先看哪里

优先看以下位置：

1. 用户数据目录下的 `logs/error.log`
2. 用户数据目录下的 `logs/startup.log`
3. 安装目录下的 `_internal` 是否缺 `imblearn`
4. `dist/FraudShield` 与安装目录结构是否完全一致

## 8. 发布建议

建议采用以下发布节奏：

1. 本机 `onedir` 自测通过
2. 安装版本机自测通过
3. 在无开发环境痕迹的测试机验证安装版
4. 测试机通过后再正式发布

不要在 `onedir` 未通过前直接切 `onefile`，否则会增加定位成本。
